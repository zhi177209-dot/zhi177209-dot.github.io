# Project Expansion Template

这个文件是给 Codex 后续续写新项目页面时使用的固定规则模版。

目标：
在现有站点结构上，以最小修改成本继续新增项目。

---

## 1. 现有站点结构约束

当前项目博客采用“首页总览 + 独立项目详情页”的结构：

- `index.html`
  只负责：
  - 首页总览区
  - 项目展示看板
  - 关于我

- `project-workspace-climate.html`
  当前示例项目详情页。

- `style.css`
  全站共用样式文件。

---

## 2. 后续新增项目时的核心规则

当用户提供“新项目资料”时，默认按以下规则工作：

1. 新建一个独立项目详情页
   命名规则：
   - `project-<slug>.html`
   - 示例：`project-smart-lock.html`

2. 只对首页 `index.html` 做最小改动
   只允许：
   - 在项目看板区域追加一个新的项目卡片
   - 必要时修改卡片数量相关的文案

3. 尽量不修改旧项目详情页
   - 不重写 `project-workspace-climate.html`
   - 不改动旧项目内容

4. 尽量不修改 `style.css`
   - 优先复用现有类名和布局
   - 只有在新项目确实需要全站通用样式支持时，才允许追加少量样式
   - 不允许大规模重构已有 CSS

5. 新项目资源单独放入 `assets/`
   - 图片、封面图、仿真图等统一放在 `assets/`
   - 文件名尽量与项目 `slug` 对应

---

## 3. 每次新增项目时的标准输出

默认应输出以下改动：

### 必做

- 新建 1 个项目详情页：
  - `project-<slug>.html`

- 修改首页：
  - 在 `index.html` 的项目看板中新增 1 个项目卡片

### 选做

- 新增资源文件：
  - `assets/<slug>-cover.png`
  - `assets/<slug>-simulation.png`

- 少量补充样式：
  - 仅在必要时对 `style.css` 追加兼容性样式

---

## 4. 新项目详情页必须沿用的页面结构

每个新项目详情页优先复用当前详情页结构：

1. 顶部导航
   - 返回首页
   - 项目简介
   - 演示/仿真
   - 核心代码
   - 关于我

2. Hero 区
   - 项目代号/分类
   - 项目标题
   - 简短项目说明
   - 技术标签
   - 进入项目 / 查看演示按钮

3. 项目简介区
   - 项目背景
   - 系统说明
   - 架构卡片 3 到 4 个

4. 演示区
   - 图片或截图展示
   - 演示说明
   - 行动按钮

5. 核心代码区
   - 2 到 3 个代码窗口
   - 标题要对应真实模块
   - 代码内容优先基于用户提供的本地项目

---

## 5. 首页项目卡片追加规则

新增项目时，只在 `index.html` 的项目看板区域追加一个新卡片。

要求：

- 不删除已有卡片
- 不重排整个首页结构
- 新卡片样式复用现有 `.project-card`
- 链接直接指向新建的 `project-<slug>.html`

卡片应包含：

- 项目状态标记
- 项目标题
- 一段简短描述
- 3 个左右的 Meta 标签
- 跳转文案

## 7. Codex 执行规则

当用户未来说“按模版继续新增一个项目”时，Codex 应默认遵守：

1. 先读取本文件。
2. 优先复用当前站点结构和现有样式。
3. 只新增：
   - 一个新的项目详情页
   - 首页中的一个新项目卡片
4. 除非必要，不改旧页面内容。
5. 除非必要，不改全局样式结构。
6. 如果用户提供了本地工程源码，代码展示区应尽量从真实源码中提取内容，而不是凭空生成。
7. 如果用户提供了图片，应优先使用用户图片，不替换成虚构资源。

---

## 8. 最小修改原则

每次新增项目时，目标是把改动控制在以下范围：

- `index.html`
  只新增一个项目卡片

- `project-<slug>.html`
  新建完整详情页

- `assets/`
  只新增该项目相关资源

- `style.css`
  能不改就不改

这是最高优先级规则。

## 9. 给 Codex 的直接提示词模版

以后可以直接把下面这段连同项目资料一起发给 Codex：

```md

注意：
- 不要重写旧项目详情页
- 不要大改 `index.html`
- 不要大改 `style.css`

以下是新项目资料：
【把新项目资料粘贴在这里】
```

---

## 10. 当前站点的推荐新增流程

以后每增加一个项目，推荐按下面流程执行：

1. 用户提供新项目资料和资源
2. Codex 读取本模版
3. Codex 新建 `project-<slug>.html`
4. Codex 在 `index.html` 中追加一个项目卡片
5. 如有需要，将新资源加入 `assets/`
6. 仅在必要时微调 `style.css`

---

这个模版的目的：
让后续扩展项目时，尽量少改当前现有 HTML 文件，只做增量式维护。
