# Project Expansion Template

这个文件是给 Codex 后续继续新增项目页面时使用的固定规则模版。

目标：
在现有站点结构上，以最小修改成本继续扩展大项目与小项目页面。

---

## 1. 当前站点结构约束

当前项目博客已经从“单一首页项目看板”扩展为“双层项目结构”：

- `index.html`
  只负责：
  - 首页总览区
  - 大项目展示看板
  - 跳转到小项目看板的入口
  - 关于我

- `small-projects.html`
  只负责：
  - 小项目页面总览区
  - 小项目展示看板
  - 页面说明

- `project-workspace-climate.html`
  当前大项目详情页示例。

- `project-stm32f103-led-mode-switch.html`
  当前小项目详情页示例。

- `style.css`
  全站共用样式文件。

当前站点的内容层级应理解为：

1. 首页 `index.html` 只收纳大项目入口
2. 小项目统一收纳在 `small-projects.html`
3. 每个项目仍然优先拆分为独立详情页 `project-<slug>.html`

---

## 2. 后续新增项目时的核心规则

当用户提供“新项目资料”时，默认先判断这是：

- 大项目
- 小项目 / 小练习 / 轻量实验 / 快速原型

然后按对应规则工作。

### 大项目规则

1. 新建一个独立项目详情页
   - 命名规则：`project-<slug>.html`

2. 只对 `index.html` 做最小改动
   - 只允许在大项目看板区域追加 1 个新卡片
   - 必要时修改大项目数量相关文案

3. 不动 `small-projects.html`
   - 除非用户明确要求调整小项目页结构

### 小项目规则

1. 新建一个独立项目详情页
   - 命名规则：`project-<slug>.html`

2. 只对 `small-projects.html` 做最小改动
   - 只允许在小项目看板区域追加 1 个新卡片
   - 必要时修改小项目数量相关文案

3. 不动 `index.html`
   - 除非用户明确要求首页同步展示此小项目入口

### 通用规则

1. 尽量不修改旧项目详情页
   - 不重写已有详情页
   - 不改动旧项目内容

2. 尽量不修改 `style.css`
   - 优先复用现有类名和布局
   - 只有在新项目确实需要全站通用样式支持时，才允许追加少量样式
   - 不允许大规模重构已有 CSS

3. 新项目资源单独放入 `assets/`
   - 图片、封面图、仿真图、视频等统一放在 `assets/`
   - 文件名尽量与项目 `slug` 对应

## 3. 每次新增项目时的标准输出

### 新增大项目时

必做：

- 新建 1 个大项目详情页
  - `project-<slug>.html`

- 修改 `index.html`
  - 在大项目看板中新增 1 个项目卡片

选做：

- 新增资源文件
  - `assets/<slug>-cover.png`
  - `assets/<slug>-simulation.png`
  - `assets/<slug>-demo.mp4`

- 少量补充样式
  - 仅在必要时对 `style.css` 追加兼容性样式

### 新增小项目时

必做：

- 新建 1 个小项目详情页
  - `project-<slug>.html`

- 修改 `small-projects.html`
  - 在小项目看板中新增 1 个项目卡片

选做：

- 新增资源文件
  - `assets/<slug>-cover.png`
  - `assets/<slug>-demo.mp4`
  - `assets/<slug>-simulation.png`
  - `assets/<slug>-serial.png`
  - `assets/<slug>-device.jpg`
  - `assets/<slug>-log.json`
  - `assets/<slug>-notes.docx`

- 少量补充样式
  - 仅在必要时对 `style.css` 追加兼容性样式

## 4. 项目详情页必须沿用的页面结构

所有新项目详情页优先复用当前已有详情页结构。

### 通用基础结构

1. 顶部导航
   - 返回对应看板页
   - 项目简介
   - 演示区
   - 核心代码区
   - 其他补充区块

2. Hero 区
   - 项目代号 / 分类
   - 项目标题
   - 简短项目说明
   - 技术标签
   - 进入项目 / 查看演示按钮

3. 项目简介区
   - 项目背景
   - 系统说明
   - 架构卡片 3 到 4 个

4. 演示区
   - 图片、截图或视频展示
   - 演示说明
   - 行动按钮
   - 如果没有视频，可直接使用 1 到 3 张真实图片完成演示区
   - 如果用户提供了串口截图、实物图、运行记录，应优先使用真实成果而不是虚构示意图

5. 核心代码区
   - 2 到 3 个代码窗口
   - 标题要对应真实模块
   - 代码内容优先基于用户提供的本地项目
   - 优先选择“主流程调度 + 核心驱动 + 显示/通信模块”这类最能解释项目闭环的代码

### 小项目详情页的推荐补充结构

如果用户提供了开发过程中的踩坑、修复过程或知识总结，小项目详情页推荐追加：

6. 问题与解决方式
   - 真实问题
   - 定位原因
   - 修复方案
   - 后续开发建议

7. 知识点补充
   - 初始化顺序
   - 中断与轮询差异
   - 其他和该练习相关的原理总结

8. 数据记录与验证区（推荐）
   - 如果用户提供了 JSON、CSV、串口记录或测量结果截图，优先整理出真实运行数据
   - 可补充记录条数、采样周期、起止时间、均值、最大最小值等
   - 该区块的作用是证明项目不是一次性截图，而是持续运行后的真实结果

如果用户没有提供这类资料，不强制添加。

## 5. 项目卡片追加规则

### 大项目卡片规则

新增大项目时，只在 `index.html` 的大项目看板区域追加一个新卡片。

要求：

- 不删除已有卡片
- 不重排整个首页结构
- 新卡片样式复用现有 `.project-card`
- 链接直接指向新建的 `project-<slug>.html`

### 小项目卡片规则

新增小项目时，只在 `small-projects.html` 的小项目看板区域追加一个新卡片。

要求：

- 不删除已有卡片
- 不重排整个小项目页结构
- 新卡片样式复用现有 `.project-card`
- 链接直接指向新建的 `project-<slug>.html`

### 卡片通用内容

卡片应包含：

- 项目状态标记
- 项目标题
- 一段简短描述
- 3 个左右的 Meta 标签
- 跳转文案

## 7. Codex 执行规则

当用户未来说“按模版继续新增一个项目”时，Codex 应默认遵守：

1. 先读取本文件。
2. 先判断新增内容属于大项目还是小项目。
3. 优先复用当前站点结构和现有样式。
4. 只新增：
   - 一个新的项目详情页
   - 对应看板页中的一个新项目卡片
5. 如果是大项目：
   - 只改 `index.html`
   - 不改 `small-projects.html`
6. 如果是小项目：
   - 只改 `small-projects.html`
   - 不改 `index.html`
7. 除非必要，不改旧页面内容。
8. 除非必要，不改全局样式结构。
9. 如果用户提供了本地工程源码，代码展示区应尽量从真实源码中提取内容，而不是凭空生成。
10. 如果用户提供了图片或视频，应优先使用用户资源，不替换成虚构资源。
11. 如果用户提供了 JSON、CSV、串口导出或其他运行记录，应优先从真实数据中提炼采样周期、统计结果和展示文案。
12. 如果用户提供了问题文档、总结文档或实验记录，应优先把其中的真实问题与解决过程整理进“问题与解决方式”区块。
13. 如果用户明确说某些内容不需要展示，例如引脚连接表或接线说明，就不要强行加入。

## 8. 最小修改原则

每次新增项目时，目标是把改动控制在以下范围：

### 新增大项目

- `index.html`
  - 只新增一个大项目卡片

- `project-<slug>.html`
  - 新建完整详情页

- `assets/`
  - 只新增该项目相关资源

- `style.css`
  - 能不改就不改

### 新增小项目

- `small-projects.html`
  - 只新增一个小项目卡片

- `project-<slug>.html`
  - 新建完整详情页

- `assets/`
  - 只新增该项目相关资源

- `style.css`
  - 能不改就不改

这是最高优先级规则。

## 10. 当前站点的推荐新增流程

以后每增加一个项目，推荐按下面流程执行：

1. 用户提供新项目资料和资源
2. Codex 读取本模版
3. 判断该项目属于大项目还是小项目
4. Codex 新建 `project-<slug>.html`
5. 如果是大项目：
   - 在 `index.html` 中追加一个大项目卡片
6. 如果是小项目：
   - 在 `small-projects.html` 中追加一个小项目卡片
7. 如有需要，将新资源加入 `assets/`
8. 如果用户提供了运行记录或问题文档，优先把它们转化为“真实数据区块”和“问题总结区块”
9. 仅在必要时微调 `style.css`

---

这个模版的目的：
让后续扩展项目时，尽量少改当前现有 HTML 文件，只做增量式维护，并明确区分大项目和小项目的展示入口。
