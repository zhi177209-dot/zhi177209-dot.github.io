STM32F103C8T6 / MINI LAB 01

按键切换 LED 流水/闪烁状态实验

PROJECT OVERVIEW

项目简介

这个项目的重点不是单独点亮某个 LED，而是把按键输入、中断响应、状态变量切换、 LED 模式执行以及串口打印调试串起来，形成一个适合嵌入式入门展示的小闭环。

系统说明

工程基于 CubeMX 生成初始化框架，GPIO 中将 PB10 配置为上升沿外部中断输入， PB3、PB4、PB5 配置为推挽输出。按键触发后，中断回调只负责改写标志位； 主循环读取标志位后决定执行哪种 LED 效果，并通过 printf 将当前模式打印到串口。

这样的结构把“快速响应”和“具体业务处理”拆开了，中断只做轻量事件通知， 而灯效与输出信息则放在主循环里完成，更符合 STM32 项目开发的规范写法。

01

Key Input

PB10 配置为 EXTI 上升沿中断，按键按下时立即进入回调函数，响应输入事件。

02

Flag Switch

回调中只切换 key10_flag 和 key10_uart，把模式状态和串口提示拆成两个轻量标志。

03

LED Output

led_off1() 实现三灯流水效果，led_off2() 实现三灯整体亮灭闪烁效果。

04

UART Feedback

通过 fputc 重定向 printf 到 USART1，模式切换后可实时在串口助手中看到文本提示。

Core/Src/led.c 
void led_off1(void)
{
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_3, GPIO_PIN_SET);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_4, GPIO_PIN_RESET);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_5, GPIO_PIN_RESET);
    HAL_Delay(500);

    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_3, GPIO_PIN_RESET);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_4, GPIO_PIN_SET);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_5, GPIO_PIN_RESET);
    HAL_Delay(500);

    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_3, GPIO_PIN_RESET);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_4, GPIO_PIN_RESET);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_5, GPIO_PIN_SET);
    HAL_Delay(500);
}

void led_off2(void)
{
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_3, GPIO_PIN_SET);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_4, GPIO_PIN_SET);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_5, GPIO_PIN_SET);
    HAL_Delay(500);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_3, GPIO_PIN_RESET);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_4, GPIO_PIN_RESET);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_5, GPIO_PIN_RESET);
    HAL_Delay(500);
}
Core/Src/stm32f1xx_it.c 
volatile uint8_t key10_flag;
volatile uint8_t key10_uart = 1;

void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin)
{
    if (GPIO_Pin == GPIO_PIN_10) {
        if (key10_flag == 0) {
            key10_flag = 1;
            key10_uart = 2;
        } else {
            key10_flag = 0;
            key10_uart = 1;
        }
    }
}

DEBUG NOTES

问题与解决方式

这部分保留你在做小项目时真正踩过的坑和修正过程，会让这个页面比单纯展示效果更有学习价值。

01

仿真按键按下程序卡死

问题出在外部中断回调 HAL_GPIO_EXTI_Callback 中调用了 HAL_Delay()。 中断服务执行时总中断关闭，而 HAL_Delay() 依赖 SysTick 计时，两者冲突后程序会直接阻塞。

删除回调内延时后卡死消失；原本想用它做按键消抖，后续改成“标志位 + 主循环轮询消抖”的合规方案。

02

实物上电误触发中断

Proteus 仿真正常，但板子上电后未按键也频繁进中断，LED 状态自行翻转。 根因是输入脚若使用 GPIO_NOPULL，实物环境下引脚会浮空，容易受到杂波干扰。

将输入配置改为 GPIO_PULLUP 后，空闲状态保持高电平，中断误触发问题被消除。

03

多文件共享中断标志变量

中断回调里修改标志，主循环里读取标志，不能通过 return 传值，也不能在多个 .c 文件里重复定义变量。

解决方式是只在一个源文件中定义全局变量，在其他需要访问的文件中通过 extern 声明，实现多文件共享。

04

后续编程建议

中断回调中禁止放延时、阻塞循环等耗时逻辑，只做置位和清零；机械按键消抖尽量放到主循环； 外部中断输入脚在实物上必须使用上拉或下拉，避免浮空。

CONCEPT NOTES

知识点补充

页面最后保留两组知识总结，既能解释这次写法，也方便以后复盘 STM32 基础外设项目。

STM32 外设标准初始化顺序

  • 开时钟：先开启 GPIO 与外设时钟，否则后续寄存器配置无效。
  • 配引脚：设置输入输出模式、上下拉、速度、复用或中断功能。
  • 配外设：如 USART、TIM、ADC、EXTI 等参数初始化。
  • 配 NVIC：设置中断优先级、使能中断通道。
  • 启外设：最后启动外设工作，形成完整初始化链路。
01

轮询

CPU 在主循环中反复查询状态，逻辑直观但会持续占用算力，实时性也受循环耗时影响。

02

中断

设备有事件时主动打断主程序，CPU 平时可继续跑主逻辑，响应更快，但中断内必须保持轻量。
